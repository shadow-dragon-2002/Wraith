# Wraith

A Windows system-tray utility that blocks physical hardware input while passing software-originated input through, so AI agents and automation can keep running while the physical keyboard and mouse are locked out.

## Language

**physical input**:
An input event where the injected flag is NOT set — the event originated from real hardware (keyboard, mouse). This is what Wraith blocks when locked.
_Avoid_: hardware input, keyboard input, real input

**synthetic input** / **injected input**:
An input event where the injected flag IS set — the event was generated by software (`SendInput`, Chrome extension, AHK, Parsec, RDP). This always passes through, regardless of lock state.
_Avoid_: programmatic input, fake input, virtual input

**locked state**:
The state in which Wraith is actively suppressing physical input. Synthetic input continues to pass through.
_Avoid_: blocked state, active state

**unlocked state**:
The state in which all input passes through normally. Normal Windows behaviour.
_Avoid_: open state, inactive state

**injected flag**:
The Windows-level bit that distinguishes physical from synthetic input. `LLKHF_INJECTED` (bit 4) for keyboard events; `LLMHF_INJECTED` (bit 0) for mouse events. Set by the kernel for all `SendInput`-originated events.
_Avoid_: synthetic flag, software flag

**hook**:
A low-level Windows callback (`WH_KEYBOARD_LL` / `WH_MOUSE_LL`) installed via `SetWindowsHookExW`. Called by Windows on every input event. Must return within ~200ms or Windows silently removes it.
_Avoid_: listener, interceptor, filter

**hook pump**:
The `GetMessageW` loop on the main thread that drives hook callbacks. If it blocks, hooks stop firing. It is the heartbeat of the application.
_Avoid_: event loop, message loop (too generic)

**panic unlock**:
An emergency unlock mechanism: hold the panic key (default: Esc) for 3 continuous seconds. Works even while locked because `GetAsyncKeyState` reads raw hardware state regardless of hook blocking.
_Avoid_: emergency unlock, failsafe

**tray icon**:
The system-tray notification area icon that shows current lock state and provides the right-click context menu.
_Avoid_: notification icon, status icon

**config**:
The INI file (`wraith.ini`) loaded once at startup, stored immutably. Holds hotkey combos, panic key, and lock-on-start flag.
_Avoid_: settings, preferences, options

**single-instance mutex**:
A named global Win32 mutex (`Global\WraithSingleInstance`) that prevents more than one Wraith process from running simultaneously.
_Avoid_: lock file, process guard

## Example Dialogue

> Dev: "What happens when the AI tool sends a click?"
> Domain expert: "The click arrives as synthetic input — the injected flag is set — so even in locked state it passes straight through the hook. Wraith never sees it as something to block."
>
> Dev: "What if someone physically grabs the mouse?"
> Domain expert: "That's physical input. The injected flag is not set. If Wraith is in locked state, the hook returns 1 and the click is consumed — it never reaches any application."
>
> Dev: "Can they use panic unlock to get in?"
> Domain expert: "Yes — that's the point. Panic unlock is the owner's failsafe. It requires physically holding Esc for 3 seconds, which Wraith detects via GetAsyncKeyState even though the hook itself is blocking the keystrokes."
